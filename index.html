HTML

  • First Computer Language: Hypertext Markup Language
  • Builds the structure of a webpage
  • Elements types in HTML: text, image, header, list which are seen elements
  • Elements types in HTML: meta related, content division, semantic divide and describe the page
  • The head element contains information about the webpage
  • Metadata tags used in head element: title, style, base, link, meta, and script
  • The body element represents the visible content shown to the user
  • The body element defines the body of the document and is an example of a semantic HTML
  • The name of an element in the body describes its content
  • Content of the body is contained between opening and closing tag with angle brackets
  • h represents heading
  • p represents paragraph or block of text
  • ul represents unordered list
  • ol represents ordered list
  • li represents list item
  • img represents image
  • The section elements are semantic elements that define a section in a document
  • section elements allow to elegantly organize data

CSS

  • Second Computer Language: Cascading Style Sheets (CSS)
  • Programming language used by developers to define how webpage conntent is presented to users
  • Defines colors, aesthetics, viual layout, and organization
  • 3 ways to sytle a webpage using CSS: inline CSS, internal CSS stylesheet, and external CSS stylesheet
  • Inline CSS styles an elemnt in an HTML file
  • An entire stylesheet can be embedded in an HTML file: internnal stylesheet
  • Preferred method is external CSS stylesheet
  • CSS rules are stored in a separate file which makes design changes easier
  • A margin indicates how much space we want around the outside of an element
  • A padding indixates how much space we want around the content inside an element
  • A CSS rule can be based on an element name
  • A selector defines the element or attributes to which the rules will apply
  • Declarations contain two important components: the CSS property and the value of the property
  • A CSS rule can also be assigned to a class attribute with a class selector
  • The period preceding card designates a class selector
  • A link element creates the connection to an external CSS file

Git

  • All Git commands can be viewed on cheat sheet provided by bootcamp. Information below documents how to create new feature branches and saving all work on GitHub
  • cd command changes working directory
  • git checkout -b feature/add-branch iS a command to create a branch and switch to that new feature branch
  • Whenever changes are made to code a new branch should be created for version control, especially when collaborating
  • When tasks are completed all changes are saved with Git (local) and pushed to GitHub (cloud)
  • git status command to check branch to ensure user is working on the correct branch
  • After making changes save on VS code and follow the steps to push to GitHub
  • git add -A command to stage changes
  • add -A command will commit all files in working branch
  • Next step will commit to the repository in GitHub: git commit -m "commit message"
  • -m command associates a message with the commit
  • The next step syncs local branch with remote branch in GitHub: git pull origin main
  • Push up all changes made to GitHub: git push origin feature/add-branch
  • Navigate to repository in GitHub to merge feature branch into main branch
  • Go to feature branch
  • Click on Pull Request at top of page
  • Click on New Pull Request
  • Select main branch first and feature branch second
  • Add a short description for pull request
  • Create Pull Request
  • Merge pull request and confirm to update main branch with new code
  • Navigate back to the main branch to see if code in main branch has been updated
